# 产物契约：output/reports/T-XXX.md

## 路径

`output/reports/T-XXX.md`

## 必需章节

- `# 代码生成报告 — T-XXX {任务标题}`
- `## 审核状态`
- `## 修改清单`
- `## 偏离说明`

## 修改清单

`## 审核状态` 必须符合 `contracts/review-status.md`。新生成或修订后的代码生成报告状态必须为 `待审核`；只有用户确认后才能改为 `已确认` 并进入对应任务的测试生成。

```markdown
| # | 文件 | 类型 | 按规格 | 说明 |
|---|---|---|---|---|
| 1 | `src/file.ts` | 修改 | ✅ | 按规格修改点 1 |
```

## 偏离条目

无偏离时写 `无`。

```markdown
### 偏离 1 — {摘要}

- **规格要求：** {原要求}
- **实际实现：** {实际实现}
- **原因：** {原因}
- **影响：** {受影响产物或行为}
```

## 编号规则

- `修改清单` 表格序号从 1 开始递增，并按规格修改点顺序排列。
- 偏离条目序号从 1 开始递增，并按序号升序排列。
- 新增偏离时追加到 `## 偏离说明` 末尾，并使用下一个序号；不要插到已有偏离顶部。

## 质量规则

- 规格中的每个修改点都必须在报告中体现。
- 审核状态必须存在，且进入测试生成前必须为 `已确认`。
- 需要用户选择的偏离必须同时写入实现阶段问题。
- 缺少本报告时，不得在 `CONTEXT.md` 中标记任务完成。
