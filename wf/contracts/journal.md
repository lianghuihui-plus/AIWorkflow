# 产物契约：JOURNAL.md

## 路径

`JOURNAL.md`

## 职责

记录工作流活动，用于跨会话恢复。

## 结构

```markdown
# 工作日志

## YYYY-MM-DD

### HH:MM — {动作}

- {摘要}
- 产物：{产物路径}
- 问题：{新增或解决的问题数量}
- 下一步：{建议}
```

## 写入规则

- 每次状态改变事务都要追加一条记录。
- 日期章节必须按 `YYYY-MM-DD` 升序排列，且同一天只能有一个日期章节。
- 当天已有日期标题时复用该章节，并把新日志追加到该日期章节末尾。
- 当天没有日期标题时，只能在文件末尾新增 `## YYYY-MM-DD` 章节，再写入日志；不得把新日期标题插入到既有日期标题和其日志之间。
- 每个日期章节必须至少包含一条 `### HH:MM — {动作}` 日志。
- 同一日期章节内的日志应按 `HH:MM` 升序追加。
- 针对具体任务或问题时，必须包含任务编号或问题编号。
- 只记录事实，不在 `JOURNAL.md` 中承载未决策内容。
