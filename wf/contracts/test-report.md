# 产物契约：output/test-report-T-XXX.md

## 路径

`output/test-report-T-XXX.md`

## 必需章节

- `# 测试报告 — T-XXX {任务标题}`
- `## 审核状态`
- `## 测试清单`
- `## 未覆盖行为`

## 测试清单

`## 审核状态` 必须符合 `contracts/review-status.md`。新生成或修订后的测试报告状态必须为 `待审核`；只有用户确认后才能改为 `已确认` 并标记对应任务测试完成。

```markdown
| # | 测试用例 | 覆盖行为 | 文件 | 状态 |
|---|---|---|---|---|
| 1 | {用例描述} | {行为} | `tests/Xxx.test.ts` | ✅ |
```

## 未覆盖行为

全部覆盖时写 `无`。

```markdown
- {行为} — {原因}
```

## 质量规则

- 覆盖规格中的正常路径、边界条件、异常路径和状态变化。
- 审核状态必须存在，且标记测试完成前必须为 `已确认`。
- 遵循现有测试框架和目录模式。
- 缺少本报告时，不得在 `CONTEXT.md` 中标记测试完成。
- 如果行为不明确导致无法安全编写测试，必须写入测试阶段问题。
