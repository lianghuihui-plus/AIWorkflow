# 产物契约：output/specs/T-XXX.md

## 路径

`output/specs/T-XXX.md`

## 必需章节

- `# T-XXX — {任务标题}`
- `## 审核状态`
- `## 任务概述`
- `## 依赖`
- `## 关键行为`
- `## 实现方案`
- `## 修改点`

## 修改点格式

`## 审核状态` 必须符合 `contracts/review-status.md`。新生成或修订后的规格状态必须为 `待审核`；只有用户确认后才能改为 `已确认` 并进入对应任务的代码实现。

```markdown
### 修改点 1：{摘要}

- **文件：** `path/to/file.ext` [NEW]
- **类型：** 新增
- **位置：** {类/函数/区域，新文件可省略}
- **内容：** {编码指令}
- **约束：** {约束}
```

## 质量规则

- `output/design.md` 中每个任务都必须有一份规格文件。
- 审核状态必须存在，且进入代码实现前必须为 `已确认`。
- 既有文件路径必须在代码仓库中确认。
- 新文件必须标注 `[NEW]`。
- 不得使用“待确认”等模糊占位；遇到不确定项应写入实现阶段问题。
- 关键行为必须来自或可追溯到 `output/analysis.md`。
