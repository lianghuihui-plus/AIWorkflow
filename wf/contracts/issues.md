# 产物契约：ISSUES.md

## 路径

`ISSUES.md`

## 必需章节

```markdown
# 待澄清

## 分析阶段
## 设计阶段
## 实现阶段
## 测试阶段
```

空章节使用 `暂无`。

## 问题条目

```markdown
### Q-001 — {问题简述}

- **问题：** {问题描述}
- **AI 建议：** {建议方案}
- **影响：** {受影响产物或模块}
- **提出：** YYYY-MM-DD {来源能力}

---

**人工决策：**
**状态：** 待决策
```

## 编号规则

- 问题编号格式为 `Q-[0-9]{3}`。
- 新问题编号从所有阶段中已有最大编号递增。
- 重复编号无效。

## 状态规则

- 未明确包含 `状态：已解决` 的问题均视为待决策。
- 已解决问题应移动到 `CHANGELOG.md` 并从 `ISSUES.md` 删除。
- 残留的已解决问题可以识别，但 `wf-status` 应提示归档。

## 写入规则

- 能力只为契约中列出的不确定项或等价阻塞歧义写问题。
- 运行时决定待决策问题是否阻塞当前动作。
- 不得编造人工决策；用户未给出前，`人工决策` 保持为空。
