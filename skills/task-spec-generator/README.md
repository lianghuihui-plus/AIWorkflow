# task-spec-generator

将技术方案收敛为逐文件逐修改点的开发规格文档，为 code-generator 提供可逐条执行的输入。支持上游设计变更后的增量更新。

## 输入

- `output/tech-design.md`（状态必须为「已审核」）

## 输出

- `output/specs/README.md` — 规格总览索引
- `output/specs/T-XXX-spec.md` — 每个任务的独立规格文件

## 使用方式

```
# 初版生成
执行 task-spec-generator → 为每个任务生成独立规格文件（状态：待审核）

# 修订（两种方式）
A. 索引修订：新增/删除任务 → 同步创建/删除对应规格文件
B. 规格修订：填写待澄清项的人工决策 → 自动收敛

# 上游设计变更后更新
执行 task-spec-generator（产物已审核时询问是否更新）→ 更新模式增量更新受影响
的 spec → 同步更新 README.md 索引 → 状态降级「待审核」→ 审核后确认
```

## 注意

- 仅在 tech-design.md 状态为「已审核」时才能执行
- 每个任务生成一个独立 spec 文件，修改点 ID 每个任务内独立编号
- 索引修订和规格修正是独立的：索引增删会同步 spec 文件，规格修订不修改索引
- 不确定的路径/签名/类型标注为「需确认」，不要猜测
- 更新模式同步更新受影响 spec 和 README.md 索引，未受影响的保持不变
- 所有待澄清项解决 → 收敛为「已审核」
