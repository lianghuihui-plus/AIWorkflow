# 工作流初始化

通过一轮交互问答收集项目信息，在指定位置创建 `workflow-xxx` 目录，生成 `workflow.yaml` 和完整目录结构。后续所有 phase 直接读配置文件，不再询问用户。

## 交互问答

按表格顺序，一问一答收集。每问一个，等待用户回答后再继续。收集完成后进入校验阶段。

| 序号 | 提示语 | 必填 | 校验规则 |
|------|--------|------|---------|
| 1 | `项目名称叫什么？` | 是 | 不能为空 |
| 2 | `开发平台是哪个？（可选：HarmonyOS / iOS / Android / Web / 其他）` | 是 | 不能为空 |
| 3 | `原始 PRD 文档路径是？` | 是 | 文件/目录必须存在 |
| 4 | `项目代码路径是？（没有的话填"无"）` | 否 | 如提供，目录必须存在 |
| 5 | `工作流根目录是？我会在该目录下创建 workflow-{项目名称}。` | 是 | 父目录不存在则自动创建，不询问 |

### PRD 路径处理（Q3 之后）

拿到 PRD 路径后，自动判断：
- **是文件**：记录该文件路径。
- **是目录**：扫描目录下所有文件（.md / .txt / .pdf），逐条记录文件路径。
- 不向用户确认，直接进入下一问。

**工作流目录命名：** 使用 `workflow-{项目名称}` 格式。如目录下已存在同名目录，提示用户确认是否覆盖或指定其他名称。
**新增问答：** 在表格中追加一行即可。

## 校验

收集完成后，逐项校验：

1. **项目名称** — 不能为空
2. **目标平台** — 不能为空
3. **PRD 文档** —
   - 文件路径：检查文件是否存在、是否可读（支持 Markdown / PDF / 纯文本）
   - 目录路径：检查目录是否存在，将该目录下所有文件视为 PRD 文档
4. **项目代码路径**（如提供）— 检查目录是否存在
5. **工作流目录位置** — 父目录不存在则直接创建，不询问用户

**校验失败 → 立即告知用户具体问题，修正后再继续。**

## 创建 workflow-xxx 目录

在用户指定的位置创建 `workflow-{项目名称}` 目录，并在其中创建固定结构：

```
{用户指定位置}/workflow-{项目名称}/
├── workflow.yaml          ← 配置文件
└── output/                ← 各阶段输出目录
```

## 生成 workflow.yaml

目录创建完成后，在根目录生成 `workflow.yaml`：

```yaml
# workflow.yaml
# 由 workflow-init 自动生成，可手动修改

project:
  name: "项目名称"
  platform: HarmonyOS
  code_path: ./my-project  # 可选，未提供时留空

prd:
  sources:
    # 按文件记录，每个文件一条。目录路径会被展开为具体文件列表。
    - docs/prd-v1.md
    - docs/other-prd/prd-v2.md
    - docs/other-prd/prd-v3.md

output:
  base_dir: output
```

初始化完成后，输出：

```

📄 已生成 workflow.yaml

下一步：执行 Phase 1 — 需求分析
  进入工作流目录，运行 prd-analyzer
```

## 规则

- 所有路径在 `workflow.yaml` 中使用相对路径（相对于 workflow-xxx 目录）
- PRD 路径为文件时，直接记录文件路径；为目录时，展开目录下所有文件，按文件逐条记录（支持 .md / .txt / .pdf 格式）
- `workflow.yaml` 生成后用户可以手动修改
- 后续 phase 的 skill 都从 `workflow.yaml` 读取配置，不再交互询问
