# 工作流初始化

通过一轮交互问答收集项目信息，在指定位置创建 `workflow-xxx` 目录，生成 `workflow.yaml` 和完整目录结构。后续所有阶段直接读配置文件，不再询问用户。

## 交互问答

按表格顺序，一问一答收集。每问一个，等待用户回答后再继续。

| 序号 | 提示语 | 必填 | 校验 |
|------|--------|------|------|
| 1 | `项目名称叫什么？` | 是 | 不能为空 |
| 2 | `开发平台是哪个？（如：HarmonyOS / iOS / Android / Web / 其他）` | 是 | 不能为空 |
| 3 | `原始 PRD 文档路径是？` | 是 | 文件或目录必须存在 |
| 4 | `项目代码路径是？（没有的话填"无"）` | 否 | 如提供，目录必须存在。填"无"则在 workflow.yaml 中 code_path 留空 |
| 5 | `工作流根目录是？我会在该目录下创建 workflow-{项目名称}。` | 是 | 父目录不存在则自动创建 |

**校验失败 → 修正后继续当前问，不跳过。**

### PRD 路径处理（Q3 之后）

拿到 PRD 路径后，自动判断：
- **是文件**：记录该文件路径。
- **是目录**：扫描目录下所有文件（.md / .txt / .pdf），逐条记录文件路径。
- 不向用户确认，直接进入下一问。

### 工作流目录命名

目录名使用 `workflow-{项目名称}` 格式。如目标位置已存在同名目录，提示用户确认是否覆盖或指定其他名称。

### 新增问答

如需追加问题，在问答表格中新增一行即可。

---

## 创建工作流目录

在用户指定的位置创建 `workflow-{项目名称}` 目录及固定结构：

```
{用户指定位置}/workflow-{项目名称}/
├── workflow.yaml          ← 配置文件（后续阶段的唯一入口）
└── output/                ← 各阶段输出目录
```

---

## 生成 workflow.yaml

目录创建完成后，在根目录生成 `workflow.yaml`。所有路径均为相对路径（相对于 workflow-xxx 根目录）。PRD 路径为目录时，展开为具体文件列表。

```yaml
# workflow.yaml
# 由 workflow-init 自动生成，可手动修改

project:
  name: "项目名称"
  platform: HarmonyOS
  code_path: ./my-project  # 未提供时留空

prd:
  sources:
    # 按文件记录，每个文件一条
    - docs/prd-v1.md
    - docs/other-prd/prd-v2.md
    - docs/other-prd/prd-v3.md

output:
  base_dir: output
```
