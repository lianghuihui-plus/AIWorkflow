# tech-designer

对已分解的任务进行技术方案设计。探索真实代码库，产出数据结构、核心流程、影响范围等技术方案。支持上游任务变更后的增量更新。

## 输入

- `output/tasks.md`（状态必须为「已确认」）
- `output/requirements.md`（用于扫描全局约束）
- `workflow.yaml` 中 `project.code_path` 指向的真实代码库

## 输出

- `output/tech-design.md` — 任务总览、数据结构变更、核心流程、入口出口、影响范围、边界场景

## 使用方式

```
# 初版生成
执行 tech-designer → 基于真实代码探索生成 tech-design.md（状态：待审核）

# 审核修订
填写待澄清项的人工决策 → 执行 tech-designer → 自动收敛

# 上游任务变更后更新
执行 tech-designer（产物已确认时询问是否更新）→ 更新模式增量更新受影响的
设计 → 状态降级「待审核」→ 审核后确认
```

## 注意

- 仅在 tasks.md 状态为「已确认」时才能执行
- 必须配置 `project.code_path` 指向真实代码库，否则无法探索代码
- 不凭空编造文件路径或类名，不确定时记录为「待澄清」
- 更新模式用上游最新内容增量更新，未受影响的设计保持不变
- 所有待澄清项解决 → 收敛为「已确认」
