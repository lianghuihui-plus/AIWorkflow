# task-decomposer

将已审核的结构化需求拆解为开发任务块，建立依赖关系并排序。支持上游需求变更后的增量更新。

## 输入

- `output/requirements.md`（状态必须为「已审核」）

## 输出

- `output/tasks.md` — 任务总览、任务详情、待澄清项

## 使用方式

```
# 初版生成
执行 task-decomposer → 生成 tasks.md（状态：待审核）

# 审核修订
填写待澄清项的人工决策 → 执行 task-decomposer → 自动收敛

# 上游需求变更后更新
执行 task-decomposer（产物已审核时询问是否更新）→ 更新模式增量更新受影响的
任务 → 状态降级「待审核」→ 审核后确认
```

## 注意

- 仅在 requirements.md 状态为「已审核」时才能执行
- 只做任务拆分和依赖排序，不涉及技术方案和实现细节
- 硬依赖才设为依赖，软依赖不设依赖
- 全局约束影响的任务会标记 `[G]`
- 更新模式用上游最新内容增量更新，未受影响的任务保持不变
- 所有待澄清项解决 → 收敛为「已审核」
