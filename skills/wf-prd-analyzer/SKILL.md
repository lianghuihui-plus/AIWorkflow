# 需求分析

扫描 `prd/` 目录下的 PRD 文件，提取结构化需求，输出 `output/analysis.md`。

> 在 workspace 目录下执行。

## 前置检查

当前目录存在 `CONTEXT.md`。不存在 → 报错终止。
`prd/` 目录存在且非空。为空 → 报错：**「prd/ 目录为空，请先放入 PRD 文件」**。

## 编号规则

| 编号类型 | 格式 | 分配规则 |
|---------|------|---------|
| PRD ID | PRD-01、PRD-02… | 按文件字母序分配 |
| 需求 ID | REQ-001、REQ-002… | 纳入和暂不纳入共用，按检出顺序 |

---

## 执行流程

### Step 1：读取 CONTEXT.md

读 `CONTEXT.md`，提取：
- 平台（「项目约束」→ `平台`）
- 代码路径（「项目约束」→ `代码仓库`，可能为"无"）
- 项目名称（标题行）

### Step 2：读取 PRD

扫描 `prd/` 目录下所有文件，**全量提取，不做任何判断和过滤**：

- 为每份 PRD 按字母序分配 ID（PRD-01、PRD-02…）
- 标注每条需求的来源 PRD ID
- 文件为空 → 在「文件清单」中标注

### Step 3：读取已有分析

`output/analysis.md` 存在 → 读取，提取已识别需求的模块归属、处理逻辑、约束条件，作为后续分流和补全的参考。

不存在 → 跳过。

### Step 4：统一分流

对每条需求逐条判断。如有代码路径：扫描项目目录结构辅助判断。

分流规则：
- 属于目标平台 → 纳入「功能需求」
- 不属于目标平台 → 纳入「暂不纳入」，标注原因
- **无法判断 → 标注 `[待确认]`**（Step 7 统一写入 ISSUES.md）

### Step 5：结构化补全

对**已纳入的功能需求**逐条补全：

- **所属模块** — 模块名/页面/组件
- **变更内容** — 按条目列出，每条一句话
- **处理逻辑** — 校验规则、判断条件、状态变化
- **约束条件** — 技术限制、接口约束
- **需求来源** — PRD ID

字段信息缺失 → 对应字段标注 `[待确认：xxx]`（Step 7 统一写入 ISSUES.md）。

### Step 6：生成 analysis.md

```markdown
# 需求分析 — [项目名称]

> 分析日期：YYYY-MM-DD
> 目标平台：[平台名称]

---

## PRD 来源

| ID | 文件 | 标题 |
|------|------|------|
| PRD-01 | xxx.md | xxx |

## 文件清单

| PRD ID | 状态 |
|--------|------|
| PRD-01 | ✅ 已分析 |
| PRD-02 | ⚠️ 空文件 |

---

## 功能需求

| ID | 所属模块 | 来源 |
|----|---------|------|
| REQ-001 | 登录模块 | PRD-01 |
| REQ-002 | 个人中心 | PRD-01 |

---

### REQ-001 — [标题]

**所属模块：** xxx

**变更内容：**
- xxx

**处理逻辑：**
- xxx

**约束条件：**
- xxx

**需求来源：** PRD-01

---

## 暂不纳入

| ID | 描述 | 不纳入原因 | 来源 |
|----|------|-----------|------|
| REQ-003 | xxx | 非目标平台 | PRD-01 |
```

### Step 7：更新工作空间文件

**更新 CONTEXT.md：**

- `## 需求概要` → 写入 2-3 句话概括
- `## 当前状态` → `阶段` 改为 `需求分析完成，待审核`，`下一步` 改为 `执行 wf-tech-designer`
**写入 ISSUES.md（Step 4/5 中所有 `[待确认]` 项）：**

在 `## 分析阶段` 下按以下格式逐条追加，编号按已有 Q-xxx 递增：

```
### Q-001 — [问题简述]

- **问题：** [具体描述]
- **AI 建议：** [建议方案]
- **影响：** [影响的产物或模块]
- **提出：** YYYY-MM-DD wf-prd-analyzer

---

**人工决策：**
**状态：** 待决策
```

**写入 JOURNAL.md：**

在文件末尾追加当天日期标题（如已存在则复用），然后追加：

```
### HH:MM — wf-prd-analyzer 完成

- 从 X 份 PRD 提取 X 条需求，X 条纳入
- 生成 output/analysis.md
- 发现问题 X 项，写入 ISSUES.md
```

- [ ] 文件清单覆盖所有 PRD，空文件已标注
- [ ] 有疑问的地方已在 ISSUES.md 或标注 `[待确认]`
