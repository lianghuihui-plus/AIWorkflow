# 工作流会话

每次使用 AIWorkFlow 工作时先执行此 skill。询问用户是新建工作流还是继续已有工作流。

## 第一步：选择模式

询问用户：

```
新建工作流还是继续已有工作流？
1. 新建 — 创建新的 workflow-xxx 目录并初始化
2. 继续 — 加载已有工作流目录继续工作
```

用户选择后进入对应分支。

---

## 分支 A：新建工作流

### A1. 交互问答

按表格顺序，一问一答收集。每问一个，等待用户回答后再继续。

| 序号 | 提示语 | 必填 | 校验 |
|------|--------|------|------|
| 1 | `项目名称叫什么？` | 是 | 不能为空 |
| 2 | `开发平台是哪个？（如：HarmonyOS / iOS / Android / Web / 其他）` | 是 | 不能为空 |
| 3 | `原始 PRD 文档路径是？` | 是 | 文件或目录必须存在 |
| 4 | `项目代码路径是？（没有的话填"无"）` | 否 | 如提供，目录必须存在。填"无"则在 workflow.yaml 中 code_path 留空 |
| 5 | `工作流根目录是？我会在该目录下创建 workflow-{项目名称}。` | 是 | 父目录不存在则自动创建 |

**校验失败 → 修正后继续当前问，不跳过。**

### A2. PRD 路径处理（Q3 之后）

拿到 PRD 路径后，自动判断：
- **是文件**：记录该文件路径。
- **是目录**：扫描目录下所有文件（.md / .txt / .pdf），逐条记录文件路径。
- 不向用户确认，直接进入下一问。

### A3. 工作流目录命名

目录名使用 `workflow-{项目名称}` 格式。如目标位置已存在同名目录，提示用户确认是否覆盖或指定其他名称。

### A4. 创建工作流目录

在用户指定的位置创建 `workflow-{项目名称}` 目录及固定结构：

```
{用户指定位置}/workflow-{项目名称}/
├── workflow.yaml          ← 配置文件（后续阶段的唯一入口）
├── AGENT.md               ← Agent 身份与约束
└── output/                ← 各阶段输出目录
```

`AGENT.md` 从 `skills/workflow-session/AGENT.md` 复制默认版本。

### A5. 生成 workflow.yaml

目录创建完成后，在根目录生成 `workflow.yaml`。所有路径均使用绝对路径。PRD 路径为目录时，展开为具体文件列表。

```yaml
# workflow.yaml
# 由 workflow-session 自动生成，可手动修改

project:
  name: "项目名称"
  platform: HarmonyOS
  code_path: ""  # 未提供时留空

prd:
  sources:
    # 按文件记录，每个文件一条
    - docs/prd-v1.md

output:
  base_dir: output

agent:
  profile: AGENT.md
```

### A6. 完成提示

```
✅ 工作流初始化完成

项目名称：XXX
目标平台：XXX
PRD 文档：X 份
项目代码：有/无
工作流目录：{用户指定位置}/workflow-xxx/

📁 已创建目录结构
📄 已生成 workflow.yaml
📄 已生成 AGENT.md

下一步：执行 prd-analyzer 开始需求分析
```

## 分支 B：继续已有工作流

### B1. 获取目录路径

询问用户：

```
工作流目录路径是？
```

### B2. 校验目录

逐项校验，失败时明确指出缺失项并中止：

1. **目录存在**：路径必须指向一个已存在的目录
2. **workflow.yaml 存在**：目录根下必须有 `workflow.yaml`
3. **workflow.yaml 可解析**：读取并解析 YAML，提取项目信息
4. **output/ 目录存在**：如不存在，提示但继续
5. **AGENT.md 存在**：目录根下必须有 `AGENT.md`

校验失败时，明确指出缺失项，让用户修正后重试。校验通过进入 B3。

### B3. 加载 AGENT.md

读取 `{工作流目录}/AGENT.md` 的完整内容，将其作为当前会话的有效约束上下文注入，确保 agent 遵守项目级编码规范和架构约定。

### B4. 展示工作流状态

加载完成后，展示当前工作流概要：

```
✅ 已加载工作流：{目录路径}

项目名称：XXX
目标平台：XXX
PRD 文档：X 份
项目代码：有/无
AGENT.md：已加载生效

等待你的指令。
```

结束。不自动启动任何下游阶段，等待用户下一步指令。
