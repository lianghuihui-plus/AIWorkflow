# workflow-session

工作流会话入口。每次开始工作流时先执行，询问新建还是继续已有工作流。

## 输入

- 新建模式：用户交互（项目名称、平台、PRD 路径、代码路径、根目录）
- 继续模式：已有工作流目录路径

## 输出

- 新建模式：创建 `workflow-{项目名称}/` 目录，含 `workflow.yaml` + `AGENT.md` + `output/`
- 继续模式：校验目录后加载 `AGENT.md` 上下文，展示工作流状态

## 使用方式

```
# 新建工作流
执行 workflow-session → 选"新建" → 按提示逐一回答 → 自动生成目录和配置

# 继续已有工作流
执行 workflow-session → 选"继续" → 提供目录路径 → 自动校验并加载上下文
```

## 注意

- 每次开始工作流工作时应先执行此 skill，确保 AGENT.md 约束生效
- 继续模式下 AGENT.md 不存在会中止
- PRD 路径为目录时自动扫描 .md / .txt / .pdf 文件
