# code-generator

基于任务规格文档逐修改点生成真实代码，直接写入项目代码仓库。每次处理一个 spec 文件，支持上游 spec 变更后的增量更新。

## 输入

- `output/specs/T-XXX-spec.md`（选定的任务规格文件）
- `workflow.yaml` 中 `project.code_path` 指向的代码仓库

## 输出

- 代码写入 `project.code_path` 下的项目文件中
- `output/generated/T-XXX-report.md` — 代码生成报告

## 使用方式

```
# 用户指定任务
执行 code-generator T-003 → 生成代码 + 报告

# 用户未指定任务
执行 code-generator → 自动定位首个未完成的任务 → 确认后生成

# 修订
填写待澄清项的人工决策 → 执行 code-generator → 修改代码 → 人工确认审核通过

# 上游 spec 变更后更新
执行 code-generator（报告已审核时询问）→ 更新模式增量更新受影响的修改点
→ 状态降级「待审核」→ 审核后确认
```

## 注意

- 每次只处理一个任务，不批量生成
- 规格文件状态必须为「已审核」才能生成，未确认会被拦截
- 不确定如何实现的修改点会被跳过，记录到报告的待澄清表中
- 代码审核需人工明确确认（如「审核 OK」），仅 CL 全部解决不足以收敛
- 更新模式对比 spec 差异，只重新生成受影响的修改点
- 已成功写入的修改点在失败时不回滚
